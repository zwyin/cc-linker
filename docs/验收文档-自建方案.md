## 5. 飞书 Bot 启动与连接

### 5.1 启动流程

| # | 场景 | 操作 | 预期 | 状态 |
|---|------|------|------|------|
| 5.1.1 | 正常启动 | `cc-bridge start` | 输出 `✅ 飞书 WebSocket 连接已建立`，`✅ cc-bridge 已启动` | ☐ |
| 5.1.2 | 获取 owner.lock | 启动时 | 获取 `~/.cc-bridge/owner.lock`，成为唯一写者 | ☐ |
| 5.1.3 | 执行启动自愈 | 启动后 | 扫描 `spool/processing/` → `pending/`，归并 session-events，修复不一致 | ☐ |
| 5.1.4 | 缺少飞书配置 | 不配置 `app_id` 启动 | 输出 `⚠️ 飞书 App ID/Secret 未配置，跳过 WSClient 连接`，不崩溃，Bot 仍可启动 | ☐ |
| 5.1.5 | 重复启动 | 已有进程运行时再次 start | 检测到 owner.lock，拒绝二次启动，提示已运行 | ☐ |
| 5.1.6 | 后台运行 | `cc-bridge start --daemon` | 输出 PID、日志路径、停止命令，终端立即返回 | ☐ |

### 5.2 优雅停机

| # | 场景 | 操作 | 预期 | 状态 |
|---|------|------|------|------|
| 5.2.1 | Ctrl+C 停止 | 在运行终端按 `Ctrl+C` | 停止接收新消息 → 等待当前处理完成（最多 10s）→ 释放 owner.lock → 退出 | ☐ |
| 5.2.2 | SIGTERM 停止 | `kill <pid>` | 行为同 Ctrl+C | ☐ |
| 5.2.3 | 处理中强制退出 | 有消息正在处理时 Ctrl+C | 超过 10s 宽限后保留 `processing` 文件，等下次启动恢复 | ☐ |
| 5.2.4 | 验证 owner.lock 释放 | 停止后 | `~/.cc-bridge/owner.lock` 文件不存在或已释放 | ☐ |

### 5.3 后台服务管理

| # | 场景 | 操作 | 预期 | 状态 |
|---|------|------|------|------|
| 5.3.1 | 后台启动 | `cc-bridge start --daemon` | 写入 PID 文件，输出日志路径，终端立即返回 | ☐ |
| 5.3.2 | 停止后台服务 | `cc-bridge stop` | SIGTERM 优雅停机，15s 后 SIGKILL 兜底 | ☐ |
| 5.3.3 | 查看服务状态 | `cc-bridge daemon status` | 显示 PID、日志、最近日志内容 | ☐ |
| 5.3.4 | 开机自启配置 | `cc-bridge daemon install` | macOS: launchd plist / Linux: systemd service | ☐ |
| 5.3.5 | 移除开机自启 | `cc-bridge daemon uninstall` | macOS: unload plist / Linux: disable service | ☐ |
| 5.3.6 | 重复安装提示 | 已配置时再次 install | 提示已存在，询问是否覆盖 | ☐ |
| 5.3.7 | 重复启动检测 | 已后台运行时再次 --daemon | 显示 PID，提示已在运行 | ☐ |
